Survey data is never clean on arrival, and skipping a consistent cleaning
process is one of the fastest ways to end up with an analysis that
quietly doesn't hold up.

## 1. Audit before you touch anything

Understand what's missing, what's duplicated, and what looks
inconsistent before making any changes.

## 2. Decide on missing-data rules up front

Agree on how missing responses are handled — exclude, impute, or flag —
before running a single test, not after seeing results you like.

## 3. Standardize before you analyze

Normalize response formats, units, and categorical labels so the same
answer is never represented two different ways.

## 4. Keep the cleaning steps auditable

Every transformation should be reproducible and reviewable, so anyone
questioning the results can see exactly what was done to the raw data.
